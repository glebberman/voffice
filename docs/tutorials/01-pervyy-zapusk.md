# Первый запуск: поднять voffice и встретиться в офисе

Это учебное руководство. Мы поднимем проект с нуля, войдём в офис двумя
пользователями и убедимся, что они видят друг друга и общаются по близости.
Ничего понимать заранее не нужно — просто выполняйте шаги по порядку.

К концу вы:

- запустите весь стек одной командой;
- войдёте в виртуальный офис;
- откроете второе окно и увидите двух аватаров на одной карте;
- отправите сообщение, которое слышно только вблизи.

Времени нужно ~10 минут (плюс первая сборка Docker-образа).

## Что понадобится

- Docker и Docker Compose;
- PHP 8.2+ и Composer (для установки зависимостей до сборки образа);
- браузер.

## Шаг 1. Получить код и зависимости

```bash
git clone git@github.com:glebberman/voffice.git
cd voffice
composer install
```

`composer install` нужен до `docker compose`, потому что образ приложения
собирается из файлов, которые устанавливает Composer.

## Шаг 2. Подготовить окружение

```bash
cp .env.example .env
php artisan key:generate
```

Файл `.env` уже настроен на работу в Docker: приложение на порту `8000`,
WebSocket-сервер Reverb на `8080`, Vite на `5173`.

## Шаг 3. Запустить всё

```bash
docker compose up -d
```

Первый запуск соберёт образ (несколько минут) и поднимет четыре контейнера:

- `app` — Laravel (порт 8000);
- `reverb` — WebSocket-сервер (порт 8080);
- `vite` — сборка фронтенда с горячей перезагрузкой (порт 5173);
- `queue` — обработчик очереди (для серверных broadcast-событий).

Проверьте, что всё поднялось:

```bash
docker compose ps
```

У всех сервисов в колонке состояния должно быть `running`. Если контейнер `vite`
только что стартовал, дайте ему минуту — при первом старте он ставит npm-пакеты.

## Шаг 4. Войти в офис

Откройте <http://localhost:8000/rooms/office> и войдите под тестовым пользователем:

- e-mail: `anya@voffice.test`
- пароль: `password`

Вы увидите карту офиса и своего аватара по имени «Аня». Походите стрелками или
клавишами WASD — персонаж двигается по клеткам, натыкаясь на столы и стены.

> Тестовые пользователи созданы заранее. Если их нет (свежая база), выполните
> `docker compose exec app php artisan db:seed` и создайте пользователей через
> экран регистрации.

## Шаг 5. Позвать второго участника

Чтобы увидеть мультиплеер, нужен второй вошедший пользователь. **Важно:** две
вкладки одного браузера делят сессию, поэтому откройте второе окно в режиме
инкогнито (или в другом браузере) и войдите там вторым пользователем:

- e-mail: `borya@voffice.test`
- пароль: `password`

Теперь в обоих окнах на карте — два аватара, и в списке «Онлайн» справа значатся
оба. Подвигайте одного: второе окно видит его перемещение в реальном времени.

## Шаг 6. Поговорить по близости

Подведите аватары вплотную друг к другу и напишите что-нибудь в поле «Сказать
вслух…». Сообщение появится над головой и в панели «Чат поблизости» у обоих.

Теперь отведите одного аватара в дальний угол и напишите снова — сообщение
**не** дойдёт до собеседника. Это и есть proximity-чат: слышно только тех, кто
в радиусе ~4 клеток.

## Если что-то пошло не так

- **Страница без стилей / белый экран** — контейнер `vite` ещё не собрал
  фронтенд. Подождите минуту и обновите страницу; проверьте `docker compose logs vite`.
- **«Подключение…» не сменяется на «В сети»** — не поднялся `reverb`. Посмотрите
  `docker compose logs reverb` — там должно быть `Starting server on 0.0.0.0:8080`.
- **Порт занят** — поменяйте `APP_PORT` / `REVERB_PORT` / `VITE_PORT` в `.env` и
  перезапустите `docker compose up -d`.
