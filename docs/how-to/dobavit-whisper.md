# Как добавить своё realtime-событие (whisper)

Эфемерные события (движение, реакции, статусы) в voffice передаются напрямую
между клиентами через whisper-события presence-канала Reverb — минуя сервер.
Добавить своё событие — значит описать его тип, отправить его при действии и
обработать на приёме. Разберём на примере события «помахать рукой» `wave`.

Всё происходит в двух файлах: [`resources/js/game/types.ts`](../../resources/js/game/types.ts)
(типы) и [`resources/js/hooks/use-office.ts`](../../resources/js/hooks/use-office.ts)
(логика). Сервер трогать не нужно.

## 1. Описать тип полезной нагрузки

В `types.ts`:

```ts
export interface WavePayload {
    id: number; // кто помахал
    to: number; // кому
}
```

## 2. Отправить событие

Внутри `useEffect` хука `useOffice` (там же, где `buzzRef.current = …`) whisper
шлётся в канал комнаты:

```ts
channel.whisper('wave', { id: me.id, to: targetId } satisfies WavePayload);
```

## 3. Обработать на приёме

В цепочку `.listenForWhisper(...)` того же канала добавьте:

```ts
.listenForWhisper('wave', (p: WavePayload) => {
    if (p.to !== userRef.current.id) {
        return;                       // не мне — игнорируем
    }
    sceneRef.current?.showReaction(userRef.current.id, '👋');
})
```

## Правила, о которых легко забыть

- **Свои события возвращаются от других вкладок того же пользователя.** Если
  событие меняет состояние (позиция, статус), в обработчике проверяйте
  `p.id === userRef.current.id` и игнорируйте своё — иначе вторая вкладка будет
  затирать состояние. Пример — `applyRemoteMove`.
- **Whisper доставляется только участникам presence-канала** (`accept_client_events_from=members`).
  Это уже настроено; неавторизованные события не пройдут.
- **Whisper не персистентен.** Кто офлайн — событие не увидит. Если нужна история
  (как у чата комнаты), это уже не whisper, а серверное broadcast-событие через
  очередь — см. `MessageSent` и [Пояснение про архитектуру](../explanation/arhitektura.md).
- **Валидируйте вход.** Реакции проверяются по белому списку эмодзи, статусы — по
  списку допустимых значений. Не доверяйте полю из чужого клиента вслепую.

## Когда whisper не подходит

Если событие должно сохраняться, влиять на БД или быть виден офлайн-пользователям —
это работа для обычного HTTP-эндпоинта и серверного `ShouldBroadcast`-события,
а не для whisper. Ориентир: **эфемерное — whisper, персистентное — сервер**.
