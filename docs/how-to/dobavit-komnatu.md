# Как добавить новую комнату

Комнаты хранятся в таблице `rooms`, а исходные карты лежат JSON-файлами в
`resources/maps/`. Сидер `RoomSeeder` читает эти файлы — они единый источник
правды. Чтобы добавить комнату, нужно создать карту и зарегистрировать её в сидере.

## 1. Создать файл карты

Создайте `resources/maps/loft.json` (slug комнаты = имя файла):

```json
{
    "rows": ["##########", "#........#", "#..DD....#", "#........#", "##########"],
    "spawn": { "x": 4, "y": 3 },
    "zones": [],
    "objects": [],
    "portals": []
}
```

Правила формата — в [справочнике «Формат карты»](../reference/format-karty.md).
Кратко: все строки `rows` одинаковой ширины, `spawn` — на проходимой клетке.

## 2. Зарегистрировать комнату в сидере

В `database/seeders/RoomSeeder.php` добавьте запись в массив `$rooms`:

```php
$rooms = [
    ['slug' => 'office', 'name' => 'Офис'],
    ['slug' => 'coworking', 'name' => 'Коворкинг'],
    ['slug' => 'loft', 'name' => 'Лофт'],   // ← новая
];
```

Сидер сам подхватит `resources/maps/loft.json` по slug.

## 3. Применить

```bash
docker compose exec app php artisan db:seed --class=RoomSeeder
```

`RoomSeeder` использует `updateOrCreate`, поэтому команду безопасно запускать
повторно — существующие комнаты обновятся, новая создастся.

## 4. Проверить

Комната появится в лобби <http://localhost:8000/rooms> и по прямой ссылке
<http://localhost:8000/rooms/loft>.

## Связать комнаты порталом

Чтобы попасть в новую комнату из офиса, добавьте портал в `resources/maps/office.json`:

```json
"portals": [
    { "x": 2, "y": 14, "to": "loft", "label": "В лофт", "tx": 4, "ty": 3 }
]
```

- `x`, `y` — где стоит портал в офисе;
- `to` — slug целевой комнаты;
- `tx`, `ty` — куда прибудет игрок в целевой комнате.

Пересидируйте (`db:seed --class=RoomSeeder`). Теперь, наступив на клетку (2, 14)
в офисе, игрок телепортируется в лофт на клетку (4, 3).

> Альтернатива через код: то же самое можно сделать администратору прямо в
> браузере — см. [Как отредактировать карту](redaktirovat-kartu.md).
