# Справочник: формат карты и тайлы

Карта комнаты — JSON-объект. Исходники лежат в `resources/maps/*.json` (имя файла =
`slug` комнаты), сидер заливает их в поле `rooms.map`. На клиенте карту разбирает
`makeMap(data)` из [`resources/js/game/map.ts`](../../resources/js/game/map.ts).

## Структура

```json
{
    "rows": ["#####", "#...#", "#####"],
    "spawn": { "x": 2, "y": 1 },
    "zones": [{ "name": "Кухня", "x1": 1, "y1": 1, "x2": 3, "y2": 1, "isPrivate": false }],
    "objects": [{ "id": "board", "type": "board", "label": "Доска", "url": "https://…", "x": 2, "y": 1 }],
    "portals": [{ "x": 1, "y": 1, "to": "coworking", "label": "В коворкинг", "tx": 3, "ty": 3 }],
    "props": [{ "id": "cabinet-1", "type": "cabinet", "x": 6, "y": 6 }]
}
```

| Поле      | Тип           | Описание                                                 |
| --------- | ------------- | -------------------------------------------------------- |
| `rows`    | `string[]`    | сетка тайлов; **все строки одинаковой ширины**           |
| `spawn`   | `{x,y}`       | стартовая клетка; **должна быть проходимой**             |
| `zones`   | `Zone[]`      | прямоугольные зоны (`x1,y1,x2,y2`, `name`, `isPrivate?`) |
| `objects` | `MapObject[]` | интерактивные объекты                                    |
| `portals` | `Portal[]`    | телепорты в другие комнаты                               |
| `props`   | `Prop[]`      | многоклеточные предметы обстановки (необязательное поле) |

Размер тайла — 32×32 px (`TILE`). Клетка `(x, y)` = символ `rows[y][x]`.

## Символы тайлов

| Символ | Тайл              | Проходимый |
| :----: | ----------------- | :--------: |
|  `.`   | пол               |     да     |
|  `#`   | стена             |    нет     |
|  `:`   | плитка кухни      |     да     |
|  `,`   | ковёр переговорки |     да     |
|  `;`   | ковёр лаунжа      |     да     |
|  `*`   | spotlight (сцена) |     да     |
|  `D`   | рабочий стол      |    нет     |
|  `K`   | кухонная стойка   |    нет     |
|  `T`   | стол переговорки  |    нет     |
|  `S`   | диван             |    нет     |
|  `P`   | растение          |    нет     |

Проходимые символы: `.` `:` `,` `;` `*`. Полный список для палитры редактора —
`TILE_CHARS` в `map.ts`.

## Зоны

Зона — прямоугольник `[x1,y1]..[x2,y2]` включительно. Если `isPrivate: true`
(например, переговорка), чат внутри зоны слышен только тем, кто в той же зоне, и
не выходит наружу — независимо от радиуса. Обычная зона нужна только для подписи.

## Spotlight `*`

Проходимый тайл. Игрок, стоящий на нём, вещает в чат на **всю комнату**, минуя
правило близости. Метод `map.isSpotlight(x, y)`.

## Объекты

| Поле   | Значения                              |
| ------ | ------------------------------------- |
| `type` | `board` \| `video` \| `map` \| `link` |
| `url`  | адрес встроенного окна (iframe)       |
| `x, y` | клетка объекта                        |

Подойдя ближе `OBJECT_RADIUS` (1.6 тайла) и нажав `X`, игрок открывает `url` в модалке.

## Порталы

| Поле     | Описание                                       |
| -------- | ---------------------------------------------- |
| `x, y`   | клетка портала в текущей комнате               |
| `to`     | slug целевой комнаты (**должна существовать**) |
| `tx, ty` | клетка прибытия в целевой комнате              |
| `label`  | подпись                                        |

## Предметы (`props`)

Тайлы `D K T S P` занимают ровно одну клетку. Предметы крупнее (стол, шкаф,
телевизор) описываются отдельным списком `props` — в карте хранится только тип и
позиция, а размеры берутся из каталога [`resources/props.json`](../../resources/props.json).
Один и тот же файл читают клиент (`resources/js/game/props.ts`) и валидация
(`MapUpdateRequest`), поэтому каталог — единственный источник правды.

| Поле   | Описание                                          |
| ------ | ------------------------------------------------- |
| `id`   | произвольный уникальный идентификатор             |
| `type` | ключ из `props.json` (`cabinet`, `desk`, `tv`, …) |
| `x, y` | **левая верхняя клетка основания**                |

Запись каталога:

```json
"cabinet": { "label": "Шкаф (высокий)", "sheet": "office/Desk, Ornate.png", "sx": 96, "sy": 0, "w": 2, "h": 1, "tall": 2 }
```

| Поле     | Описание                                               |
| -------- | ------------------------------------------------------ |
| `sheet`  | путь внутри `public/assets/lpc/`                       |
| `sx, sy` | левый верхний угол спрайта в тайлсете (px)             |
| `w, h`   | размер **основания** в тайлах — эти клетки непроходимы |
| `tall`   | сколько тайлов спрайта поднимается **над** основанием  |

### Высокие предметы и проход за ними

Спрайт режется на две части. Нижние `h` тайлов (основание) рисуются **под**
игроками и блокируют проход. Верхние `tall` тайлов рисуются **над** игроками, но
проход не блокируют: персонаж проходит за предметом и частично скрывается им.

Чтобы игрок не терялся за высоким спрайтом, вокруг **своего** персонажа
вырезается овал прозрачности: верхний слой маскируется по инверсии, а копия того
же слоя показывается внутри овала с `alpha ≈ 0.32`. Подробнее —
[docs/explanation/resheniya.md](../explanation/resheniya.md).

### Горизонтальные стены в два тайла

Стена `#` занимает одну клетку логически, но рисуется в два тайла: над каждой
стеной дорисовывается «верхушка» (`map.isWallCrown(x, y)` — клетка проходима, а
под ней `#`). Верхушка попадает в тот же верхний слой, что и высокие предметы:
за ней можно ходить, и её так же прорезает овал.

## Ограничения размера

Максимум — **512×512 тайлов** (262 тыс. клеток). Предел выбран как «максимум для
адекватной работы»: карта такого размера весит ~258 КБ сырого JSON (~15–30 КБ в
gzip, тайловые строки жмутся отлично), а рендер и редактор не зависят от размера
карты, потому что оба показывают только видимую область.

Проверяется в `MapUpdateRequest` (`map.rows` ≤ 512 элементов, каждая строка
≤ 512 символов) и в `MAX_MAP_SIZE` из `map.ts`.

## Инварианты (проверяются и в PHP, и в JS)

- все строки `rows` одинаковой ширины;
- периметр карты — сплошная стена (`#`);
- `spawn` на проходимой клетке;
- объекты и порталы в границах карты;
- `portals.*.to` — существующая комната;
- `props.*.type` — ключ из `resources/props.json`;
- основание предмета целиком внутри карты, а высокой части хватает места сверху
  (`y - tall >= 0`).

Серверная проверка — `app/Http/Requests/MapUpdateRequest.php`; клиентская —
`tests/js/map.test.ts` и `tests/js/props.test.ts`.
